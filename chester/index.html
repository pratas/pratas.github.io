Introduction

What is CHESTER ?

CHESTER is a probabilistic tool to map and visualize relative singularity regions. The version 1 has been published in RECPAD Conference in October 2015, winning the best poster/paper. Version 2 enables to store data using one bit precision (instead of a byte), automatic setup of Bloom filters, inverted repeats optimization and enlargement of painted (segmented) regions. It is a version optimized to cope with large ancient genomes.

It answers the following questions:
  • Whose regions are present in multiple sequences but absent from others?
  • Can CHESTER visualize those regions (creating a image)? (yes)
  • Can CHESTER use really low memory (trading space by precision)? (yes)
  • Does it setup Bloom filters automatically? (yes)

How do I get CHESTER ?

Clone our repository and run CMake command: 
git clone https://github.com/pratas/chester.git
cd chester/src/
cmake .
make
Alternatively to CMake, run: 
git clone https://github.com/pratas/chester.git
cd chester/src/
cp Makefile.nix Makefile
make

Usage

To see the possible options type 
./CHESTER-map
or 
./CHESTER-map -h
These will print the following options: 
Usage: CHESTER-map <OPTIONS>... [FILE]:<...> [FILE]:<...>
CHESTER-map: a tool to map relative singularity regions  
The (probabilistic) Bloom filter is automatically set.

  -v                       verbose mode,
  -a                       about CHESTER,
  -t <value>               threshold [0.0;1.0],
  -w <value>               window size,
  -u <value>               sub-sampling,
  -s <value>               bloom size,
  -i                       use inversions,
  -p                       show positions/words,
  -k <value>               k-mer size (up to 30),

  [rFile1]:<rFile2>:<...>  reference file(s),
  [tFile1]:<tFile2>:<...>  target file(s).

The reference files may be FASTA, FASTQ or DNA-SEQ,
while the target files may be FASTA or DNA-SEQ.
